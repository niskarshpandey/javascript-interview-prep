# 12 - Scope and Hoisting Important Questions

## 1. What is scope?

### Answer

Scope defines where variables can be accessed.

---

## 2. Difference between global and block scope?

### Answer

Global:
Accessible everywhere.

Block:
Accessible only inside `{}`.

---

## 3. Why avoid var?

### Answer

Because var is not block scoped.

Can create unexpected bugs.

---

## 4. What is lexical scope?

### Answer

Child scope can access parent scope variables.

---

## 5. What is hoisting?

### Answer

JavaScript moves declarations to memory before execution.

---

## 6. Why does function declaration work before definition?

### Answer

Because function declaration is fully hoisted.

---

## 7. Why does function expression fail before definition?

### Answer

Because variable is hoisted but initialization happens later.
